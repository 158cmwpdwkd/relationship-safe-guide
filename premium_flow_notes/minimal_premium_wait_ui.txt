1. 수정한 파일 목록
- `imweb/imweb premium-survey.html`

2. 수정한 함수 목록
- `submitSurvey()`

3. 실제 변경한 DOM/CSS/JS 요약
- DOM: 결과 버튼 아래에 `#rl-wait-note` 안내 문구 1줄을 추가했다.
- CSS: `#rl-paid .rl-wait-note` 스타일을 추가해 버튼 아래에 작게 표시되도록 했다.
- JS: `submitSurvey()` 시작 시 버튼을 그대로 비활성화한 상태에서 버튼 텍스트를 `리포트 생성 중...`으로 바꾸고 안내 문구를 표시하도록 했고, 오류 시 안내 문구를 다시 숨기도록 했다.

4. 기존 흐름을 건드리지 않았는지 확인
- `PAID_ENDPOINT` 호출 흐름은 그대로다.
- 기존 `GENERATE_ENDPOINT` 호출과 성공 시 redirect 흐름도 그대로 유지했다.
- polling, status API, 백엔드 호출 방식은 추가/변경하지 않았다.

5. 테스트 포인트
- 결과 보기 클릭 직후 버튼이 즉시 비활성화되는지 확인
- 버튼 문구가 `리포트 생성 중...`으로 바뀌는지 확인
- 버튼 아래 안내 문구가 표시되는지 확인
- 생성 성공 시 기존과 동일하게 redirect 되는지 확인
- 생성 실패 시 안내 문구가 사라지고 버튼이 다시 활성화되는지 확인
