1. 결제 취소 후 발생하던 실제 문제 정의

- 무료 리포트에서 프리미엄 결제 버튼을 눌렀다가 취소하면 premium wrapper가 `결제가 필요합니다` 상태로 남는다.
- 이때 premium 쪽은 정리되어도, 사용자가 무료 리포트로 자연스럽게 돌아갈 경로가 명확하지 않았다.
- 결과적으로 premium 실패가 무료 리포트 재열람 동선을 끊는 문제가 있었다.

2. premium wrapper에서 수정한 항목

- 파일: `imweb/premium_report.html`
- 수정 내용
  - `NOT_PAID / STATE / ERROR / PROCESSING` 상태에서 iframe 래퍼 높이를 `0px`로 명시해 잔여 프레임 높이가 남지 않게 했다.
  - `무료 리포트 다시 보기` 보조 링크를 state UI에 추가했다.
  - free return path 해석 로직 추가
    - `rl_free_report_return_url`
    - `rl_free_report_token`
    - `document.referrer`
    순으로 복귀 경로를 결정한다.
  - `NOT_PAID` 진입 시 `payment.cancel_or_not_paid` 로그 추가
  - 복귀 링크 표시 시 `return.path.resolve`, `return.cta.ready`
  - 복귀 링크 클릭 시 `return.cta.click`

3. 무료리포트 복귀가 막히던 원인

- free report -> premium CTA 진입 시, “원래 무료 리포트 URL”을 premium 쪽이 알 수 있는 형태로 저장하지 않고 있었다.
- 그래서 결제 취소 후 premium wrapper 안에서는 `결제가 필요합니다` 상태는 보여도, 사용자를 원래 무료 리포트 화면으로 되돌리는 안정적인 경로가 없었다.

4. 수정 후 정상 동작 정의

- 무료 리포트에서 premium CTA 클릭
- 결제창 취소
- premium wrapper는 iframe 없이 `결제가 필요합니다` 상태 UI만 표시
- polling 없음
- frame/session/timer 정리 유지
- 상태 UI 아래 `무료 리포트 다시 보기` 링크 노출
- 사용자는 저장된 free return URL로 돌아가 무료 리포트를 다시 열람 가능

5. 테스트 시나리오

- 무료리포트 -> 결제 버튼 -> 취소
  - free wrapper에서 `premium.cta.entry` 로그 확인
  - premium wrapper에서 `payment.cancel_or_not_paid` 로그 확인
  - iframe이 숨겨지고 `frameWrapEl` 높이가 0인지 확인

- 취소 후 무료리포트 재열람
  - premium wrapper의 `무료 리포트 다시 보기` 링크 클릭
  - 원래 free report URL로 복귀하는지 확인
  - free report iframe이 정상 로드되는지 확인

- 취소 후 premium 재진입
  - 무료 리포트 페이지에서 premium CTA를 다시 눌러 결제 진입 가능한지 확인
  - 이전 premium 실패 상태가 free report 렌더를 오염시키지 않는지 확인

6. 남은 리스크 1개

- 현재 premium/free wrapper 파일 자체의 한글 인코딩이 일부 깨진 상태라, 브라우저에서 문구가 정상인지 별도 확인이 필요하다. 다만 이번 라운드의 핵심 수정은 복귀 URL 저장/복원과 state 화면 안정화다.
