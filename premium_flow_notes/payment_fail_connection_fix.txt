1. 실제 수정한 서버/결제 연결 위치
- app/routes_payments.py
  - PAYMENT_FAIL_URL 상수 추가
  - _render_client_return_page() 수정
  - /api/payments/inicis/close 는 exit 경유 스크립트를 그대로 사용
  - script load fail, auth fail, approve fail 계열은 모두 /api/payments/inicis/exit 로 모인 뒤 payment-fail 로 이동

2. 기존에 404를 만들던 경로
- 존재하지 않거나 준비되지 않은 /payment-cancel
- 실패 시 free 복귀 URL을 직접 계산하는 중간 흐름

3. 지금 payment-fail로 연결되도록 바꾼 경로
- script load fail
  - /api/payments/inicis/exit?mode=fail&code=INICIS_SCRIPT_LOAD_FAIL
- 결제창 close
  - /api/payments/inicis/close
- 인증/승인 실패
  - /api/payments/inicis/exit?mode=fail&code=...
- 위 exit 페이지는 최종적으로 https://reconnectlab.co.kr/payment-fail 로 이동

4. close/fail/exit 흐름 정리 방식
- close/fail 는 직접 free URL로 보내지 않는다.
- 먼저 /api/payments/inicis/exit 또는 close 응답에서 mode/code/msg 를 sessionStorage 에 저장한다.
- 그 다음 payment-fail URL 로 단일 이동한다.
- free return URL / free token 은 payment-fail 페이지에서 읽어서 복귀 버튼에 사용한다.

5. 테스트 방법
- 무료리포트에서 결제 버튼을 누른다.
- 결제창에서 취소한다.
- premium wrapper NOT_PAID 대신 https://reconnectlab.co.kr/payment-fail 로 들어오는지 확인한다.
- payment-fail 에서 무료리포트 다시 보기 버튼이 보이는지 확인한다.
- 버튼 클릭 후 원래 무료리포트가 정상 렌더되는지 확인한다.
