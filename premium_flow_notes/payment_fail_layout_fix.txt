1. body -> layout-root 구조 변경 설명
- 기존 `payment-fail` 페이지는 `body` 자체가 `display:flex`, `align-items:center`, `justify-content:center`를 맡고 있었습니다.
- 아임웹 HTML 위젯 환경에서는 `body`가 전체 브라우저 뷰포트가 아니라 위젯 영역 기준으로만 동작할 수 있어 중앙 정렬 기준이 흔들릴 수 있습니다.
- 이번 수정에서는 `body`에서 중앙 정렬 책임을 제거하고, 새 최상위 래퍼 `.layout-root`가 전체 콘텐츠의 수평/수직 정렬을 담당하도록 바꿨습니다.

2. 제거한 스타일 목록
- `body.display: flex`
- `body.flex-direction: column`
- `body.align-items: center`
- `body.justify-content: center`
- `body.overflow: hidden`

3. 중앙 정렬 기준 변경 내용
- `site-header`, `main.page`, `footer-brand`를 모두 `.layout-root` 내부로 이동했습니다.
- `.layout-root`는 아래 역할을 가집니다.
  - `min-height: 100vh`
  - `width: 100%`
  - `display: flex`
  - `flex-direction: column`
  - `align-items: center`
  - `justify-content: center`
- `.page`의 기존 `max-width: 520px`는 그대로 유지해 카드 폭과 디자인 값은 바꾸지 않았습니다.

4. 아임웹에서의 동작 결과
- 이제 중앙 정렬 기준이 body가 아니라 `layout-root` 전체 폭 기준이므로, 아임웹 위젯 안에서도 카드가 가운데로 모이도록 구조를 바꿨습니다.
- body는 배경/패딩 역할만 유지하므로, 위젯 컨텍스트에서 body 정렬이 깨져도 메인 카드 정렬에는 직접 영향이 가지 않게 했습니다.

5. 남은 리스크 (iframe width 관련)
- 아임웹 위젯 바깥 래퍼가 자체 좌측 padding, 좁은 width, 또는 transform 계열 스타일을 주는 경우 최종 위치는 여전히 외부 컨테이너 영향이 남을 수 있습니다.
- 다만 현재 파일 내부 기준으로는 body 정렬 의존성을 제거했기 때문에, 같은 증상이 반복되더라도 원인은 아임웹 외부 래퍼 CSS 쪽으로 더 좁혀집니다.
