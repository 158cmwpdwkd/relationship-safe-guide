1. 생성/수정한 아임웹 HTML 파일명
- imweb/imweb payment-fail.html

2. 페이지 역할 요약
- 결제 취소/실패 예외 흐름을 premium wrapper 내부 NOT_PAID 상태와 분리한다.
- 사용자는 이 페이지에서 결제 실패 안내를 보고, 버튼 1개로 원래 무료리포트로 복귀한다.
- iframe, polling, postMessage 로직은 포함하지 않는다.

3. 무료리포트 복귀 경로 결정 규칙
- 1순위: localStorage 의 rl_free_report_return_url
- 2순위: localStorage 의 rl_free_report_token 기반 https://reconnectlab.co.kr/report?token=...
- 3순위: https://reconnectlab.co.kr/
- document.referrer 는 사용하지 않는다.

4. 정리하는 premium/payment 임시 상태 목록
- localStorage: rl_premium_report_token
- sessionStorage: rl_payment_last_mode
- sessionStorage: rl_payment_last_code
- sessionStorage: rl_payment_last_msg
- 유지:
  - rl_free_report_return_url
  - rl_free_report_token

5. 브라우저에서 확인할 로그 이벤트
- payment_fail.enter
- payment_fail.return_path.raw
- payment_fail.return_path.selected
- payment_fail.return_cta.click

6. 테스트 방법
- 결제 취소 후 reconnectlab.co.kr/payment-fail 로 진입한다.
- 제목, 설명, 무료리포트 다시 보기 버튼이 노출되는지 확인한다.
- 버튼 클릭 시 저장된 free return URL 또는 free token 기준 URL로 이동하는지 확인한다.
- 복귀 후 무료리포트가 정상 렌더되고 premium 실패 상태가 다시 노출되지 않는지 확인한다.
