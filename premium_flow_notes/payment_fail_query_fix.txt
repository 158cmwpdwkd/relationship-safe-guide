1. 기존에 free_return_url이 reject되던 이유
- query `free_return_url`가 들어와도 validation 이 불필요하게 엄격해서 reject 되고 있었다.
- 특히 경로/host 검증이 fallback 로직과 섞이면서, 실제 정답인 `https://reconnectlab.co.kr/report?token=...` 도 reject 후 home fallback 으로 떨어질 여지가 있었다.

2. decode/parse 로직 수정 내용
- `free_return_url` 를 읽은 뒤 `decodeURIComponent` 적용
- `new URL(decoded, window.location.origin)` 으로 파싱
- 로그 추가:
  - payment_fail.return_path.query_url_raw
  - payment_fail.return_path.query_url_decoded
  - payment_fail.return_path.query_url_parsed

3. validation 완화 내용
- strict pathname 비교 제거
- `pathname.includes("/report")` 방식으로 변경
- 조건:
  - hostname === reconnectlab.co.kr 또는 same-origin
  - pathname 에 `/report` 포함
  - token 존재
- 위 조건 만족 시 무조건 selected URL 로 채택

4. 최종 selected URL 규칙
- 1순위: valid 한 query `free_return_url`
- 선택된 URL 에 `from_payment_fail=1` 을 붙여 CTA href 로 세팅
- query URL 이 valid 하면 storage/fallback 은 보지 않음

5. 테스트 시나리오
- 결제 취소 후 payment-fail 진입
- `payment_fail.return_path.query_url_raw` 확인
- `payment_fail.return_path.query_url_selected` 확인
- `payment_fail.return_cta.href` 확인
- 무료리포트 다시 보기 클릭
- 홈 fallback 없이 free report URL 로 이동하는지 확인
