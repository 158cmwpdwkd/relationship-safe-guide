1. fallback이 문제를 다시 발생시키던 이유
- popup 차단 시 `window.top.location.href` 또는 `window.location.href` 로 `/pay/start` 를 여는 fallback 이 남아 있었다.
- 이 fallback 때문에 결제 페이지가 다시 메인 free DOM 을 덮었고, `/pay/start` 문맥 오염 문제가 재발할 수 있었다.

2. fallback 제거 방식
- `imweb/imweb free-report.html` 에서 결제 시작 fallback 을 제거했다.
- popup 실패 시에는 더 이상 top navigation 을 하지 않는다.
- 대신 로그를 남기고 alert 로 팝업 허용을 안내한다.

3. popup-only 구조 설명
- 결제 시작은 `window.open(payUrl, "_blank")` 만 사용한다.
- 성공 시:
  - `free.payment.open_new_window`
  - `free.payment.open_target`
- 실패 시:
  - `free.payment.open_failed`
  - `free.payment.popup_blocked`
  - 사용자 안내 alert
- 메인 free 페이지는 어떤 경우에도 결제 URL 로 이동하지 않는다.

4. 테스트 결과
- popup 허용 상태
  - 결제창이 새 창에서 열리고 free DOM 은 유지되어야 한다.
- popup 차단 상태
  - 메인 free 페이지 주소가 바뀌지 않아야 한다.
  - `/pay/start` 가 메인 창에 열리지 않아야 한다.
  - 팝업 허용 안내가 떠야 한다.
