What changed
1. Central premium state orchestration added
- New file: app/services/premium_report.py
- Added resolve_premium_state(report_token, order_id=None, db)
- Added build_entry_response(...)
- Added prepare_premium_preview_or_raise(...)
- Added run_premium_pipeline(...)
- Added submit_paid_survey_and_run_pipeline(...)

2. Premium entry flow added
- New endpoint: GET /api/premium/entry?token=...&order_id=...
- routes_premium.py now uses resolve_premium_state() to return:
  - state
  - next_action
  - next_url
  - user_message

3. Paid survey now auto-runs premium pipeline
- POST /api/premium/survey/paid now saves the survey and immediately runs:
  - generate payload
  - markdown generation
  - HTML render
  - report save/finalize
- Added alias endpoint: POST /api/premium/survey/submit
- No internal HTTP self-call is used
- Internal service functions are called directly

4. Existing generate/finalize/status routes now share state validation
- routes_premium.py now routes validation through resolve_premium_state()
- Blocking conditions mapped consistently:
  - INVALID_REPORT_TOKEN -> 404
  - REPORT_ORDER_MISMATCH -> 400
  - NOT_PAID -> 403
  - NEED_SURVEY -> 404 PAID_SURVEY_NOT_FOUND on generate/finalize path

5. /r/{token} access control updated
- routes_report.py now uses resolve_premium_state()
- Behavior:
  - INVALID_REPORT_TOKEN -> 404 page
  - NEED_SURVEY -> redirect to premium survey page
  - NOT_PAID with known order -> 403 block page
  - READY with premium html -> render premium html
  - PROCESSING -> 202 processing page
  - otherwise fallback to free report html

6. premium_pipeline.py refactored for reuse
- Existing function kept:
  - prepare_premium_report_payload(...)
- Added reusable functions:
  - generate_premium_report_artifacts(...)
  - finalize_premium_report_record(...)
- These are now used by orchestration service instead of duplicating logic in routes

7. Schema additions
- Added to app/schemas.py:
  - PremiumStateOut
  - PremiumEntryOut
  - SurveySubmitOut
  - PremiumResolvedState
  - PremiumNextAction

8. Test updates
- Existing tests updated to patch the new pipeline function location
- Existing paid survey / generate / finalize / status tests were adjusted to the new flow
- Added orchestration tests:
  - entry NOT_PAID
  - entry NEED_SURVEY
  - entry READY
  - /r/{token} blocks NOT_PAID
  - /r/{token} redirects NEED_SURVEY
