1. 현재 무료 리포트 출력 구조 분석 요약

- 무료 리포트 wrapper 파일:
  - imweb/imweb free-report.html
- 무료 리포트 백엔드 HTML 생성 파일:
  - app/report.py
- 무료/프리미엄 공통 조회 엔드포인트:
  - GET /r/{token}
  - app/routes_report.py

- 현재 무료 리포트 출력 구조
  1) 아임웹 wrapper 페이지가 token을 읽는다.
     - query `?token=...`
     - path `/r/{token}`
     - localStorage `rl_report_token`
  2) wrapper가 iframe src를 백엔드로 설정한다.
     - `API_BASE + "/r/" + encodeURIComponent(token) + "?v=" + Date.now()`
  3) 백엔드가 완성된 HTML 문자열을 반환한다.
     - 무료 리포트: `app/report.py -> make_report_html(...)`
     - 프리미엄 리포트: `reports.html` 컬럼 내용 반환
  4) iframe 내부 HTML이 postMessage로 높이를 wrapper에 전달한다.
     - type: `RCL_REPORT_HEIGHT`
  5) 무료 리포트는 추가로 risk 상태도 보낸다.
     - type: `RCL_RISK_LEVEL`
  6) wrapper는 loading / error / iframe / CTA 박스를 토글한다.

- 무료 리포트 wrapper에서 실제 확인된 패턴
  - token 저장 키: `rl_report_token`
  - iframe id: `rcl-report-frame`
  - 높이 이벤트: `RCL_REPORT_HEIGHT`
  - risk 이벤트: `RCL_RISK_LEVEL`
  - fallback: token 없으면 error 표시
  - nav / footer / premium CTA 는 wrapper가 담당
  - 실제 report body 렌더링은 iframe 내부 백엔드 HTML이 담당

- 무료 리포트 디자인 토큰
  - 배경 톤:
    - `--bg: #0F1626`
    - `--bg2: #1A2540`
    - `--bg3: #222E4A`
  - 포인트 컬러:
    - `--rose: #D4916C`
    - `--rose-lt: #EAB89A`
    - `--teal: #4ECDC4`
    - `--amber: #FFB347`
    - `--red: #FF6B6B`
  - 텍스트:
    - `--txt: #F0ECE8`
    - `--sub: #9AAABB`
  - 카드 스타일:
    - dark card + 얇은 border + 14px radius
  - spacing:
    - body padding / section padding / section 간격이 작고 조밀함
  - typography:
    - `Noto Sans KR`
  - wrapper tone:
    - 진한 navy 배경
    - rose gradient CTA
    - free report body는 card 중심 세로 스택

2. 프리미엄 출력 구조 추천안

- 추천안: A. 기존처럼 iframe 유지

3. 추천안 선택 이유

- 현재 구조와의 호환성
  - 이미 `/r/{token}`가 완성 HTML을 반환하는 구조가 존재한다.
  - 무료 리포트 wrapper도 iframe 기반이다.
  - 프리미엄도 같은 계약을 유지하면 경로/배포/임웹 연동이 가장 단순하다.

- 아임웹 수정량
  - 최소 수정이다.
  - 기존 free-report wrapper를 복제해서 premium-report wrapper로 바꾸면 된다.
  - 새 프론트 런타임이나 DOM 삽입 로직이 필요 없다.

- CSS 충돌 위험
  - iframe 유지가 가장 낮다.
  - 아임웹 본문 CSS와 premium report CSS가 섞이지 않는다.
  - 기존 무료 리포트도 같은 이유로 iframe을 쓰고 있다.

- 유지보수성
  - 백엔드 HTML 렌더러가 free/premium 모두 소유권을 가진다.
  - wrapper는 token 읽기, entry 확인, iframe src 세팅, height sync만 담당하면 된다.
  - 상태 분기를 백엔드 기준으로 통일하기 쉽다.

- 결론
  - 프리미엄도 “아임웹 wrapper + iframe + 백엔드 HTML 반환”을 유지하는 것이 현재 프로젝트와 가장 맞다.
  - HTML 직접 삽입(B안)은 아임웹 CSS 충돌, XSS/CSP 관리, wrapper 복잡도 증가 때문에 비추천이다.

5. 파일별 수정 포인트 요약

- app/services/reporting/premium_renderer.py
  - 무료 리포트와 유사한 토큰/카드/spacing 톤으로 CSS 재정렬
  - hero / metric cards / markdown content / footer 구성
  - processing / error / blocked / need-survey 상태용 HTML 렌더러 추가
  - iframe height postMessage 추가
  - wrapper가 읽을 state postMessage는 선택적으로 `RCL_PREMIUM_STATE` 추가 가능

- app/routes_report.py
  - `/r/{token}`에서 `resolve_premium_state()` 결과를 기준으로 분기
  - READY: premium html 반환
  - PROCESSING: backend-generated processing HTML 반환
  - NOT_PAID: backend-generated blocked HTML 반환
  - NEED_SURVEY: backend-generated survey CTA HTML 반환 또는 302
  - INVALID: backend-generated invalid HTML 반환
  - free fallback은 “order가 없는 순수 무료 token”일 때만 유지

- imweb/imweb premium-report.html
  - free-report wrapper를 기반으로 복제
  - 진입 시 `GET /api/premium/entry?token=...` 호출
  - entry state에 따라:
    - INVALID_REPORT_TOKEN -> wrapper error
    - NOT_PAID -> 결제 CTA / 또는 iframe에 block page 표시
    - NEED_SURVEY -> survey page로 이동 버튼
    - PROCESSING -> iframe `/r/{token}` 표시
    - READY -> iframe `/r/{token}` 표시
  - iframe height 수신은 기존 `RCL_REPORT_HEIGHT` 재사용

6. 백엔드 렌더링 구조 설계

- 유지할 함수
  - `render_premium_report_html(markdown_text, metrics=None) -> str`

- 추가할 함수
  - `render_premium_state_html(state: str, report_token: str, next_url: str | None = None, user_message: str | None = None) -> str`
  - `render_processing_html(...)`
  - `render_blocked_html(...)`
  - `render_need_survey_html(...)`
  - `render_invalid_html(...)`

- 역할 분리
  - `routes_report.py`
    - 상태 판정
    - 어떤 HTML을 반환할지 결정
  - `premium_renderer.py`
    - 상태별 HTML 문자열 생성
    - premium 본문 HTML 문자열 생성
  - `premium_pipeline.py`
    - markdown -> final premium html 호출만 유지

- 상태 분기 원칙
  - 외부 진입은 token 1개
  - 실제 상태 판정은 `resolve_premium_state()` 기준
  - `/r/{token}`도 같은 resolver 사용

7. 아임웹 wrapper 연동 방식

- wrapper가 맡아야 할 역할
  - token 읽기
  - localStorage 저장
  - `GET /api/premium/entry?token=...` 호출
  - entry 응답에 따라 CTA / iframe / error UI 토글
  - iframe height 반영

- wrapper가 맡지 말아야 할 역할
  - premium 본문 렌더링
  - 상태 판정 로직 복제
  - report body CSS 소유
  - markdown/html 직접 조합

- token 읽는 방식
  - free wrapper와 동일
  - query -> path -> localStorage 순

- iframe src 세팅 방식
  - `API_BASE + "/r/" + encodeURIComponent(token) + "?v=" + Date.now()`

- free wrapper 복제 사용 여부
  - 맞음
  - 가장 적은 수정으로 가능
  - nav / loading / error / iframe height sync / footer 패턴을 그대로 재사용 가능

- 최소 수정 버전
  - free wrapper를 복제
  - premium CTA 블록 제거
  - entry API 호출 추가
  - state별 버튼 문구만 premium용으로 조정

def _short_summary(text: str) -> str:
    normalized = " ".join((text or "").split())
    if not normalized:
        return ""
    return normalized[:90].rstrip() + ("..." if len(normalized) > 90 else "")


def _report_shell(inner_html: str, *, state: str | None = None) -> str:
    premium_state_event = ""
    if state:
        premium_state_event = f'window.parent.postMessage({{ type: "RCL_PREMIUM_STATE", state: "{escape(state)}" }}, "*");'

    return f"""<!doctype html>
<html lang="ko">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover" />

def _render_metrics_html(metrics: dict | None) -> str:
    cards = metrics.get("cards") if isinstance(metrics, dict) else []
    if not cards:
        return ""

    card_map = {}
    for card in cards:
        if isinstance(card, dict) and card.get("id"):
            card_map[str(card["id"])] = card

    card_order = [
        "relationship_distance",
        "emotional_temperature",
        "contact_timing",
        "current_risk_signal",
    ]
    bar_order = [
        "recovery_conditions",
        "emotional_stability",
    ]

    card_html = []
    for card_id in card_order:
      card = card_map.get(card_id)
      if not card:
          continue
      score = max(0, min(100, int(card.get("score") or 0)))
      card_html.append(
          f'''
          <section class="metric-card" data-metric-id="{escape(card_id)}">
            <div class="metric-title">{escape(str(card.get("title") or ""))}</div>
            <div class="metric-value-row">
              <div class="metric-value">{escape(str(card.get("value_text") or ""))}</div>

from .db import SessionLocal

router = APIRouter()


def load_report_and_session_by_token(token: str):
    db = SessionLocal()
    try:
        rep = db.query(Report).filter(Report.report_token == token).first()
        if not rep:
            raise HTTPException(status_code=404, detail="Invalid token")

        sess = db.query(UserSession).filter(UserSession.sid == rep.sid).first()
        if not sess:
            raise HTTPException(status_code=404, detail="Session not found")

        return rep, sess
    finally:
        db.close()


def resolve_report_html(token: str) -> HTMLResponse:
    db = SessionLocal()
    try:
        rep, sess = load_report_and_session_by_token(token)
        state = resolve_premium_state(report_token=token, db=db)

        if state.state == "INVALID_REPORT_TOKEN":
            return HTMLResponse(
                render_premium_state_html(
                    state="INVALID_REPORT_TOKEN",
                    report_token=token,
                    user_message=state.user_message,
                ),
                status_code=404,
            )

        if state.state == "NOT_PAID" and state.order_id is not None:
            return HTMLResponse(
                render_premium_state_html(
                    state="NOT_PAID",
                    report_token=token,
                    next_url=state.next_url,
                    user_message=state.user_message,
                ),
                status_code=403,
            )

        if state.state == "NEED_SURVEY":
            return HTMLResponse(
                render_premium_state_html(
                    state="NEED_SURVEY",
                    report_token=token,
                    next_url=state.next_url,
                    user_message=state.user_message,
                ),
                status_code=200,
            )

        if state.state == "PROCESSING":
            return HTMLResponse(
                render_premium_state_html(
                    state="PROCESSING",
                    report_token=token,
                    next_url=state.next_url,
                    user_message=state.user_message,
                ),
                status_code=202,
            )

        if state.state == "READY" and (rep.html or "").strip():
            return HTMLResponse(rep.html)

        return HTMLResponse(
            make_report_html(
                risk_level=sess.risk_level,
                impulse=sess.impulse_index,
                fear_type=sess.fear_type,
            )
        )
    finally:
        db.close()

async def view_report(token: str):
    return resolve_report_html(token)

10. 최종 판정

- 가장 적은 수정으로 가능한지
  - 가능
  - 기존 무료 구조인 “아임웹 wrapper + iframe + 백엔드 HTML 반환”을 그대로 유지하면 된다.

- 추가 확인이 필요한 점
  - 현재 `/r/{token}`에서 “무료 token인데 order가 없는 경우”와 “프리미엄 token인데 NOT_PAID인 경우”를 어떻게 구분할지 운영 데이터 기준 확인 필요
  - `GET /api/premium/entry`의 next_url이 아임웹 실제 premium-survey 페이지 URL과 일치하는지 확인 필요
  - premium survey 제출 후 redirect_url / next_url 계약이 wrapper와 survey page에서 일관적인지 확인 필요

  4) READY 상태에서 premium HTML이 free report보다 시각적으로 확장되지만 브랜드 톤은 유지되는지
  5) INVALID/NOT_PAID/NEED_SURVEY/PROCESSING/READY 전 상태를 실제 token으로 검증했는지
