1. 무한 스크롤 원인 분석

- wrapper 원인
  - `READY`가 아닌 상태에서도 iframe 문서가 살아 있으면 내부 `RCL_REPORT_HEIGHT` 메시지가 계속 부모로 올라올 수 있다.
  - wrapper가 상태와 무관하게 height 메시지를 반영하면, 로딩/에러/결제 유도 화면에서도 iframe 높이가 흔들릴 수 있다.
  - polling은 `PROCESSING` 전용이어야 하는데, 상태 전환 시 iframe과 polling을 같이 정리하지 않으면 반복 렌더처럼 보일 수 있다.

- renderer 원인
  - 내부 HTML은 `DOMContentLoaded`, `load`, `fonts.ready`, `ResizeObserver`, `resize`에서 높이를 다시 계산한다.
  - 같은 높이나 미세한 차이도 반복 전송되면 parent가 계속 `height`를 다시 적용하면서 루프처럼 보일 수 있다.

2. wrapper 수정 포인트

- 파일: `imweb/premium_report.html`
- 이번 최신본 기준 반영 내용
  - `HEIGHT_DEBOUNCE_MS`: `120 -> 160`
  - `HEIGHT_MIN_DELTA`: `18 -> 24`
  - `currentState` 변수 추가
  - `showLoading()`
    - `currentState = "PROCESSING"`
    - `resetFrame()` 호출
  - `showError()`
    - `currentState = "ERROR"`
  - `showState()`
    - `currentState = "STATE"`
  - `showFrame()`
    - `currentState = "READY"`
  - `window.message` listener
    - `currentState !== "READY"`면 `RCL_REPORT_HEIGHT` 무시

- 결과
  - `NOT_PAID / NEED_SURVEY / INVALID / ERROR / PROCESSING` 상태에서는 iframe 높이 메시지를 반영하지 않음
  - `PROCESSING / STATE / ERROR` 진입 시 기존 iframe `src`가 바로 제거됨
  - `READY`일 때만 iframe이 살아 있고 height 반영이 동작함

3. renderer 수정 포인트

- 파일: `app/services/reporting/premium_renderer.py`
- 현재 유지 중인 반영 내용
  - `lastMeasuredHeight` 추가
  - 최소 전송 간격 강화: `180ms`
  - same height 또는 `8px` 이하 차이는 재전송 금지
  - 초기 burst 축소: `0 / 160 / 420 / 1000`
  - `ResizeObserver`에 `140ms` debounce 적용

- 결과
  - height postMessage가 과도하게 중복 전송되지 않음
  - 렌더 직후 필요한 재측정은 유지하면서 루프성 잔떨림을 줄임

4. 최종 수정 파일 목록

- `imweb/premium_report.html`
- `app/services/reporting/premium_renderer.py`

5. 수정된 핵심 코드

- wrapper
```js
const HEIGHT_DEBOUNCE_MS = 160;
const HEIGHT_MIN_DELTA = 24;

let currentState = "BOOT";

function showLoading(title, copy) {
  currentState = "PROCESSING";
  hideAll();
  resetFrame();
  loadingEl.style.display = "flex";
}

function showError() {
  currentState = "ERROR";
  hideAll();
  resetFrame();
  errorEl.style.display = "flex";
}

function showState(title, copy, ctaLabel, ctaHref) {
  currentState = "STATE";
  hideAll();
  resetFrame();
  stateEl.style.display = "flex";
}

function showFrame(token) {
  currentState = "READY";
  hideAll();
  frameWrapEl.style.display = "block";
  if (activeFrameToken !== token) {
    frameEl.classList.remove("loaded");
    frameEl.style.height = "600px";
    frameEl.src = API_BASE + "/r/" + encodeURIComponent(token);
    activeFrameToken = token;
  }
}

window.addEventListener("message", function (event) {
  const data = event.data;
  if (!data || data.type !== "RCL_REPORT_HEIGHT") return;
  if (typeof data.height !== "number" || data.height <= 0) return;
  if (currentState !== "READY") return;

  const target = Math.min(Math.max(data.height + 72, 720), 14000);
  applyFrameHeight(target);
});
```

- renderer
```js
var lastSentHeight = 0;
var lastMeasuredHeight = 0;
var lastSentAt = 0;
var rafId = 0;
var resizeTimer = 0;

function sendHeight(force) {
  var now = Date.now();
  var h = calcHeight();
  if (!h) return;
  if ((now - lastSentAt) < 180) return;
  if (Math.abs(h - lastSentHeight) <= 8) return;
  if (!force && Math.abs(h - lastMeasuredHeight) <= 8) return;
  lastMeasuredHeight = h;
  lastSentHeight = h;
  lastSentAt = now;
  window.parent.postMessage({ type: "RCL_REPORT_HEIGHT", height: h }, "*");
}

function scheduleHeightBursts() {
  requestHeight(true);
  setTimeout(function() { requestHeight(true); }, 160);
  setTimeout(function() { requestHeight(true); }, 420);
  setTimeout(function() { requestHeight(true); }, 1000);
}

if (window.ResizeObserver) {
  var ro = new ResizeObserver(function() {
    if (resizeTimer) {
      clearTimeout(resizeTimer);
    }
    resizeTimer = setTimeout(function() {
      resizeTimer = 0;
      requestHeight(false);
    }, 140);
  });
  ro.observe(document.documentElement);
  if (document.body) {
    ro.observe(document.body);
  }
}
```

6. 어떤 상태에서 polling이 허용되는지

- 허용: `PROCESSING`
- 금지: `INVALID_REPORT_TOKEN`, `NOT_PAID`, `NEED_SURVEY`, `READY`, fetch error

현재 wrapper 동작:
- `PROCESSING`: `scheduleRecheck(token)`
- 그 외 상태: `clearPolling()` 후 종료

7. 최종 검증 방법

1. `NOT_PAID`
   - wrapper가 결제 유도 UI만 보여주는지 확인
   - iframe `src`가 제거되는지 확인
   - 추가 polling이 없는지 확인

2. `NEED_SURVEY`
   - 설문 이동 UI만 보여주는지 확인
   - iframe `src`가 제거되는지 확인
   - polling이 없는지 확인

3. `PROCESSING`
   - 로딩 UI만 보이고 iframe은 비어 있는지 확인
   - 약 3초 간격 entry 재조회만 발생하는지 확인

4. `READY`
   - iframe이 한 번만 로드되는지 확인
   - `24px` 이하 높이 변화는 무시되는지 확인
   - 문서 하단까지는 정상적으로 늘어나되 미세한 height 흔들림이 반복되지 않는지 확인
